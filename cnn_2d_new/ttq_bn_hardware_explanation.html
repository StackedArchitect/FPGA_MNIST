1 What is TTQ?

Trained Ternary Quantization constrains every weight to one of three values — but unlike simple ternary (TWN), the positive and negative magnitudes are learned independently.

TWN (Ternary Weight Network)

Weights ∈ {-1, 0, +1}
Symmetric — positive and negative weights have equal magnitude.
Result: acc += activation or acc -= activation

TTQ (Trained Ternary Quantization)

Weights ∈ {-Wn, 0, +Wp}
Asymmetric — Wp and Wn are learned per layer.
Result: Wp × Σ(pos activations) − Wn × Σ(neg activations)

Layer Chaining

Each layer's done signal is wired to the next layer's rstn. While Conv1 is running, Conv2/FC1/FC2 are held in reset. When Conv1 finishes, pool1_done goes high — releasing Conv2. This creates a sequential, zero-overlap pipeline: Conv1 → Conv2 → FC1 → FC2.

S_IDLE — Initialisation

Resets all counters: filter_idx=0, conv_out_row=0, conv_out_col=0, conv_pos=0, ch_cnt=0, kr_cnt=0, kc_cnt=0. Zeros pos_acc and neg_acc. Transitions immediately to S_CONV_COMPUTE.

S_CONV_COMPUTE — Split Ternary Accumulation

This is the MAC core. Each cycle, the pipeline stage reads:

p1_data ← data_in[ch * H*W + (row+kr)*W + (col+kc)]   // activation
p1_code ← weights[filter * TAP_COUNT + tap_idx]         // ternary code

One cycle later (pipe_s1_valid), the accumulation fires:

case (p1_code)
  2'b01: pos_acc <= pos_acc + p1_data;   // weight = +1 → add to positive sum
  2'b11: neg_acc <= neg_acc + p1_data;   // weight = -1 → add to negative sum
  default: ;                              // weight =  0 → skip (free sparsity!)
endcase

The nested loop (kc → kr → ch) iterates over all TAP_COUNT kernel taps. For Conv1: 1×3×3 = 9 taps. For Conv2: 4×3×3 = 36 taps.

Key insight: No multiplier is used! The MAC is reduced to conditional add/subtract because weights are ternary.

S_CONV_DRAIN — Pipeline Flush (2 cycles)

The 1-stage pipeline means the last tap's data is still in p1_data/p1_code when S_CONV_COMPUTE exits. The drain state collects it:

// drain_cnt=0: collect last pipeline tap into pos_acc or neg_acc
// drain_cnt=1: accumulators are final → transition to S_CONV_SCALE

S_CONV_SCALE — TTQ Wp/Wn Multiply + Bias (1 cycle) 🔑

This is the key TTQ step — the only state that uses DSP multipliers in the MAC path:

biased_reg ← (Wp × pos_acc_q16) >>> 16
            − (Wn × neg_acc_q16) >>> 16
            + bias[filter_idx]

pos_acc_q16 = pos_acc[31:0] — the full Q16.16 value of the positive sum.

>>> 16 = arithmetic right shift — converts the 64-bit multiply product back to Q16.16.

Hardware cost: 2 DSP48 slices (one for Wp×pos, one for Wn×neg), fired once per output position — not per tap. For Conv1 with 9 taps per position, this is 9× more efficient than full-precision MAC.

S_CONV_BN — Folded BatchNorm (1 cycle)

Applies the pre-computed (folded) BatchNorm as a single multiply-add:

bn_product_reg ← (bn_scale[filter] × biased_q16) >>> 16

Where biased_q16 = biased_reg[31:0] — Q16.16 value from S_CONV_SCALE.

The bn_shift is added combinationally: bn_result = bn_product_reg + bn_shift[filter]

BN folding: During training, BN computes γ(x−μ)/√(σ²+ε) + β. At export, this is folded into scale = γ/√(σ²+ε) and shift = β − μ×scale. Hardware just does scale × x + shift.

S_CONV_STORE — ReLU + Store to Conv Buffer (1 cycle)

if (activation_function && bn_result > 0)
    conv_buf[conv_pos] ← bn_result;    // ReLU pass
else if (activation_function)
    conv_buf[conv_pos] ← 0;            // ReLU clamp negative → 0

Resets pos_acc=0, neg_acc=0 for the next output position.

If more positions remain: → S_CONV_COMPUTE (loop back).
If all positions done for this filter: → S_POOL_COMPARE (start pooling).

S_POOL_COMPARE — Max Pool 2×2 (4 cycles per pool position)

Iterates over the 2×2 window in the conv buffer, tracking the maximum:

if (conv_buf[pool_in_row * CONV_OUT_W + pool_in_col] > cur_max)
    cur_max ← conv_buf[...];

After all 4 elements compared → S_POOL_STORE.

S_POOL_STORE — Store pooled result

Writes cur_max to the output array: data_out[filter*POOL_OUT_POS + pool_pos]

If more pool positions: → S_POOL_COMPARE
If all pool positions done AND more filters: reset counters, → S_CONV_COMPUTE for next filter
If all filters done: → S_DONE

S_DONE — Signal completion

Asserts done=1. This releases the next layer's reset.

S_IDLE

Resets neuron_idx=0, input_idx=0, w_addr=0, pos_acc=0, neg_acc=0. → S_FILL

S_FILL — Pipeline Prime

Loads the first weight code and input data into the pipeline registers (p1_code, p1_data). Advances input_idx and w_addr. → S_MAC

S_MAC — Ternary Accumulate

Same split accumulation as Conv — exactly the same case statement on p1_code:

2'b01 (+1): pos_acc += p1_data
2'b11 (-1): neg_acc += p1_data
2'b00 ( 0): skip

Iterates LAYER_NEURON_WIDTH cycles (FC1: 239 inputs, FC2: 71 inputs — including zero padding).

Weight codes are read from BRAM ROM (w_rom), initialized via $readmemh.

S_DRAIN

Same 2-cycle pipeline flush as Conv. Collects last tap, then → S_SCALE.

S_SCALE — TTQ Scaling + Bias

biased_reg ← (Wp × pos_acc[31:0]) >>> 16
            − (Wn × neg_acc[31:0]) >>> 16
            + bias[neuron_idx]

If HAS_BN=1 → S_BN. If HAS_BN=0 (FC2) → S_STORE directly.

S_BN — BatchNorm (FC1 only)

bn_product_reg ← (bn_scale[neuron] × biased_reg[31:0]) >>> 16
final_result = bn_product_reg + bn_shift[neuron]

FC2 has HAS_BN=0, so this state is skipped and final_result = biased_reg directly.

S_STORE — ReLU + Output

Applies ReLU (if activation_function=1): clamps negative to 0.

Stores result to data_out[neuron_idx]. Resets pos_acc=0, neg_acc=0.

If more neurons: neuron_idx++ → S_FILL (loop back).
If last neuron done: → S_DONE

S_DONE

Asserts counter_donestatus=1. For FC2, this is the final completion signal.

Bottom Line

TTQ+BN is the recommended approach. It loses only 1.07% accuracy from the full-precision baseline while compressing weights 15.7×, eliminating all per-tap multipliers, and needing only 3 DSP48 slices per layer. Hardware simulation confirms <0.01% numerical error versus the software model.
